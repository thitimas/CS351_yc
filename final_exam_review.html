1. Algorithm Analysis & Asymptotic Notation

Key idea: describe how the running time grows with input size n, ignoring constants and lower-order terms.

Three notations

  • O(f(n)) — upper bound: the algorithm runs in at most this much time (worst case).
  • Ω(f(n)) — lower bound: the algorithm needs at least this much time.
  • Θ(f(n)) — tight bound: same upper and lower asymptotic order.

Common orders, slowest growth → fastest

O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

Example: Insertion sort's inner-loop count is 1 + 2 + … + (n-1) = n(n-1)/2 = Θ(n²). We say "insertion sort is O(n²) worst case, Ω(n) best case (already sorted)."

2. Search

Linear search

Scan the array left to right, return index when found.

Binary search (requires sorted array)

Compare to middle; recurse into the half that may contain the key.

function binarySearch(A, key):
  lo ← 0; hi ← n − 1
  while lo ≤ hi:
    mid ← (lo + hi) / 2
    if A[mid] = key  return mid
    else if key < A[mid]  hi ← mid − 1
    else  lo ← mid + 1
  return NOT_FOUND
CaseTimeSpace (iter)Space (rec)
All casesO(log n)O(1)O(log n)

k-th largest element (selection problem)

  • Sort + index: O(n log n)
  • Min-heap of size k: O(n log k) — push, pop when size > k, root is the k-th largest
  • Quickselect (partition-based): O(n) average, O(n²) worst

Binary heap as an array

Root at A[1]; for node i:

  • Parent = i / 2
  • Left child = 2i, Right child = 2i + 1
Heap property: in a max-heap, parent ≥ children; in a min-heap, parent ≤ children. Build-heap runs in O(n) (tight analysis), not O(n log n).

Hash table — collision handling

  • Chaining: each slot holds a linked list of entries.
  • Open addressing: linear / quadratic probing, double hashing.

Insertion sort (analysis)

for j = 2 to n:
  key ← A[j]
  i ← j − 1
  while i > 0 and A[i] > key:
    A[i+1] ← A[i]
    i ← i − 1
  A[i+1] ← key
Example (sort 5 1 4 2 8): after j=2: 1 5 4 2 8 · j=3: 1 4 5 2 8 · j=4: 1 2 4 5 8 · j=5: 1 2 4 5 8.

Merge sort recurrence

T(n) = 2T(n/2) + Θ(n) → by the Master Theorem, Θ(n log n).

Quick sort partitioning (Lomuto)

function partition(A, lo, hi):
  pivot ← A[hi]
  i ← lo − 1
  for j = lo to hi − 1:
    if A[j] ≤ pivot:
      i ← i + 1; swap A[i], A[j]
  swap A[i+1], A[hi]
  return i + 1
Worst case O(n²) when pivots are always the smallest/largest — mitigated by random pivot or median-of-three.

Counting sort

Stable, linear time, but only for small-integer keys (k = range of keys). Not a comparison sort.

5. Common Algorithm Patterns

Two pointers

Use one pointer from each end (or both from start) to scan a sorted/array structure in O(n).

Example — pair sum: given sorted array, find i, j with A[i] + A[j] = target. Move i++ if sum too small, j-- if too big.

Sliding window

Maintain a window [L, R] and a running aggregate; slide R right, shrink from L when invariant breaks. Most problems run in O(n).

Example: longest substring without repeating characters; max sum of size-k subarray.

Sock merchant (hashing for counting)

Walk the array once, hash-count each value. Pairs of color c = ⌊count[c] / 2⌋. Time O(n), space O(unique).

6. Greedy Algorithms

Greedy idea: at each step, take the locally optimal choice and never reconsider. Works only when the problem has optimal substructure + the greedy-choice property.

Coin change (US denominations {1, 5, 10, 25})

At each step pick the largest coin ≤ remaining amount.

Example: 31¢ → {25} (6 left) → {25, 5} (1 left) → {25, 5, 1}.
Caveat: greedy is not optimal for arbitrary denominations (e.g. {1, 3, 4} for 6¢ → greedy gives {4,1,1}=3 coins, optimal is {3,3}=2 coins).

Activity selection

Pick the activity with the earliest finish time, drop activities that overlap, repeat.

Sort by finish time: O(n log n). Then one pass: O(n).

Huffman coding

Build an optimal prefix code: repeatedly merge the two lowest-frequency nodes into a new internal node with combined frequency. Use a min-heap (priority queue).

StepCost
Build heap of n leavesO(n)
n − 1 merges, each two extract-min + one insertO(n log n)
TotalO(n log n)

Fractional knapsack

Sort items by value/weight ratio descending; take items greedily, take a fraction of the last one to fill capacity. O(n log n).

Note: greedy works for fractional knapsack only. 0/1 knapsack needs DP — see next section.

7. Dynamic Programming

When to use DP: the problem has (1) optimal substructure (optimal solution built from optimal subproblem solutions) and (2) overlapping subproblems (the same subproblems are solved many times).

Memoization vs Tabulation

  • Memoization — top-down recursion + cache.
  • Tabulation — bottom-up table filling in dependency order.

Fibonacci — the DP intro

// naive recursion: T(n) = T(n-1)+T(n-2)+O(1) ≈ O(φⁿ) — exponential!
function fib(n):
  if n < 2: return n
  return fib(n-1) + fib(n-2)

// memoized: O(n) time, O(n) space
memo[0] = 0; memo[1] = 1
function fibMemo(n):
  if memo[n] defined: return memo[n]
  memo[n] ← fibMemo(n-1) + fibMemo(n-2)
  return memo[n]

// bottom-up: O(n) time, O(1) space
a ← 0; b ← 1
for i = 2 to n:
  (a, b) ← (b, a + b)
return b

Rod cutting

Cut a length-n rod into pieces to maximize revenue, given price p[i] for a piece of length i.

Recurrence: r(n) = max1≤i≤n (p[i] + r(n − i)), with r(0) = 0.

Time O(n²), space O(n).

0/1 Knapsack

Items have weight w[i], value v[i]; capacity W; either take item i or don't.

Recurrence: K[i][w] = max(K[i−1][w], v[i] + K[i−1][w − w[i]]).

Time and space O(n·W) — pseudo-polynomial.

Example: items {(w=2, v=3), (w=3, v=4), (w=4, v=5)}, W=5 → optimal = items 1&2, value 7.

Longest Common Subsequence (LCS)

Given strings X (length m) and Y (length n), find the longest subsequence common to both.

Recurrence: if X[i] = Y[j] then c[i][j] = c[i−1][j−1] + 1, else c[i][j] = max(c[i−1][j], c[i][j−1]).

Time and space O(m·n).

Example: LCS of ABCBDAB and BDCAB is BCAB (length 4).

BFS — Breadth-First Search

Explore neighbors level by level using a queue. Finds shortest paths in unweighted graphs.

Time O(V + E), Space O(V).

DFS — Depth-First Search

Explore as deep as possible before backtracking; recursive or stack-based.

Time O(V + E). Tracks discovery time d[u] and finish time f[u] (3-color WHITE/GRAY/BLACK).

Topological sort (on a DAG)

A linear ordering of vertices such that for every edge u → v, u appears before v.

DFS-based: when a vertex turns BLACK, push it onto a stack. Output is the stack reversed (i.e. sort by decreasing finish time).

Time O(V + E). Detects a cycle if DFS finds a back edge (gray → gray).

Dijkstra's algorithm (single-source shortest path)

Requires non-negative edge weights. Greedy: at each step extract-min by tentative distance, relax outgoing edges.

function dijkstra(G, source):
  dist[source] ← 0;  dist[v] ← ∞ for all other v
  PQ ← min-heap with (0, source)
  while PQ not empty:
    (d, u) ← extract-min
    if d > dist[u]: continue   // skip stale entry
    for each edge (u, v, w):
      if dist[u] + w < dist[v]:
        dist[v] ← dist[u] + w
        prev[v] ← u
        insert (dist[v], v) into PQ
ImplementationTime
Binary heap PQO((V + E) log V)
Fibonacci heapO(E + V log V)

Minimum Spanning Tree (MST)

Connect all vertices with the minimum total edge weight, no cycles.

  • Kruskal's: sort edges by weight, add an edge if its endpoints are in different components (Union-Find / DSU). O(E log E).
  • Prim's: grow the tree from a starting vertex, always adding the cheapest edge crossing the cut. O(E log V) with binary heap.
Disconnected graphs: Kruskal produces a minimum spanning forest — one tree per connected component.

9. K-Nearest Neighbors (KNN)

Idea: a "lazy learner" — no training. To classify a new point, find the K closest training points and take the majority vote.

Algorithm

function KNN-Classify(training_set, x_query, k):
  for each (x_i, label_i) in training_set:
    d_i ← distance(x_query, x_i)
  sort by d_i ascending
  neighbors ← first k entries
  return majority label among neighbors

Distance metrics (most common)

  • Euclidean (L₂): √Σ(xᵢ − yᵢ)² — straight line.
  • Manhattan (L₁): Σ|xᵢ − yᵢ| — city-block.
  • Chebyshev (L∞): max|xᵢ − yᵢ| — largest single-axis gap.

Runtime

PhaseTimeSpace
Train (just store)O(1)O(n·d)
Predict (naive)O(n·d + n·log n)O(n)

n = training size, d = feature dimensions. Distance computation is O(n·d); sorting to find the K nearest is O(n·log n).

Choosing K: small K = low bias, high variance (noisy); large K = high bias, low variance (oversmooth). Pick K by cross-validation.

Master Theorem (quick form)

For T(n) = a · T(n/b) + f(n) with a ≥ 1, b > 1:

  • If f(n) = O(n^(log_b(a) − ε)) → T(n) = Θ(n^log_b(a))
  • If f(n) = Θ(n^log_b(a)) → T(n) = Θ(n^log_b(a) · log n)
  • If f(n) = Ω(n^(log_b(a) + ε)) (and regularity) → T(n) = Θ(f(n))
Merge sort: T(n) = 2T(n/2) + Θ(n) → case 2 → Θ(n log n).
